22/03/2026

Como um incentivo para cuidar mais do meu próprio blog (e manter esse site vivo) decidi tentar criar uma extensão para me ajudar a não ter preguiça de TODA VEZ que eu for fazer um blog pra cá eu ter que criar uma cópia do template de blog que eu uso, ir adicionando as tags bonitinho, pipipi popopo...

Criei um código para transcrever um arquivo de texto para a template HTML de blogs que eu uso.

(A partir de agora pode ficar um pouquinho técnico)

Funciona quase como um parser de texto markdown p/ HTML, a diferença é que não é markdown, eu só detecto o texto puro do arquivo .txt e detecto quando há quebra de linhas: quando só tem uma quebra de linha (\n), o programa lê como uma nova linha no mesmo parágrafo, igual vai acontecer agora:

[Exemplo]

Mas quando o parser detecta que 2 linhas (ou mais) foram puladas, ele corta para outro parágrafo, igual vai acontecer agora:

[Exemplo 2]

E é isso. Quando o texto passa no código eu também passo como parâmetro o título do blog e a data (que pode ser pega por padrão) e passo o arquivo de destino. Depois disso eu posso ir no arquivo e fazer a mudança que eu quiser.

Talvez, pra um futuro, eu pense em uma forma de ter um detector de tag IMG ou Anchor ou algo mais complexo assim, mas acho que esse trabalho eu posso fazer no arquivo que o parser cospe, não é tão trabalhoso assim.

Um próximo passo também vai ser implementar o código do parser para ser usado como um comando da CLI. To aprendendo a usar o Click do python pra isso.
